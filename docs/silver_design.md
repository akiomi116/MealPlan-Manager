# スマート献立マネージャー：シルバー運用設計書 (Silver Friendly Design)

本プロジェクトにおいて、独居老人ユーザーの認知負荷を最小限に抑え、直感的に利用できるように考慮した設計要素の一覧です。

## 1. 認証 (Authentication)
**「パスワードを覚えなくていい」**
- **Magic Link (マジックリンク) 採用**:
    - メールアドレスを入力し、届いたメールのリンクをクリックするだけでログイン完了。
    - 複雑なパスワードの記憶・管理が不要。「パスワードを忘れた」というトラブル自体を排除しました。

## 2. ユーザーインターフェース (UI/UX)
**「見やすく、押しやすく、迷わない」**
- **Big Button & Card UI**:
    - ボタンやクリック領域を通常の1.5〜2倍のサイズに設計。
    - マウス操作やタップの精度が落ちていても、誤操作しにくい「カード型」の選択肢を採用。
- **High Contrast Mode (ハイコントラスト)**:
    - 加齢による水晶体の黄変や視力の低下に対応。
    - 設定一つで「白背景・黒文字・太い枠線」のくっきりモードに切り替え可能。
- **Voice Guide (音声ガイド)**:
    - 画面の文字を読むのが辛いときのために、AIが「次はこれをしてください」「献立ができました」と音声で状況を案内。
    - 視覚だけでなく聴覚でも情報を補完します。

## 3. インタラクション (Interaction)
**「書かない、入力しない」**
- **Voice Input (音声入力)**:
    - 「牛乳を追加して」「やっぱり人参はやめて」など、キーボード入力の代わりに声で指示が可能。
- **QR Code Handoff**:
    - PCの大きな画面で見たいが、写真はスマホで撮りたいというニーズに対応。
    - ログイン操作なしで、QRコードを写すだけでスマホが「カメラ」として機能し、PCへ自動連携されます。

## 4. コンセプト (Concept)
**「管理しない」**
- **No Inventory (在庫管理の廃止)**:
    - 「何がいくつ残っているか」をちまちま入力・管理する作業は、認知負荷が高すぎます。
    - **「使う直前に写真を撮るだけ」** という、記憶に頼らないフローに一本化し、システムの使用ハードルを極限まで下げました。
